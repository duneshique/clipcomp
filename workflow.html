How it works

6단계로 완성되는
자동 압축 파이프라인

클립보드 변화 감지부터 알림까지, ClipComp는 사용자 개입 없이 모든 과정을 처리합니다.

STEP 01

클립보드 변화 감지

1초마다 클립보드 상태를 폴링합니다. clipboard info의 MD5 해시를 이전 값과 비교해 변경 여부를 판단합니다.

# 클립보드 타입 목록 해시 CLIP_INFO=$(osascript \ -e 'clipboard info') CURRENT=$(echo "$CLIP_INFO" | md5)
STEP 02

PNG 타입 확인

clipboard info 결과에서 PNGf 타입 존재 여부를 먼저 확인합니다. 텍스트·JPEG 복사는 이 단계에서 완전히 무시됩니다.

PNGf ✓
다음 단계 진행
그 외
무시, 대기 복귀
STEP 03

이미지 추출

AppleScript로 클립보드의 PNG 데이터를 임시 파일에 저장합니다. pbpaste는 텍스트 전용이므로 osascript를 직접 사용합니다.

set data to clipboard as «class PNGf» write data to /tmp/clip_in.png
STEP 04

pngquant 압축

256색 팔레트로 변환하며 품질 80–95를 유지합니다. speed 1 설정으로 최대 압축률을 적용합니다.

pngquant 256 \ --quality=80-95 \ --speed 1 \ clip_in.png \ -o clip_out.png
STEP 05

클립보드 교체

압축된 이미지를 클립보드에 덮어씁니다. 이후 PREV_COUNT를 즉시 갱신하여 자기 자신을 재감지하는 루프를 방지합니다.

# 교체 후 루프 방지 set clipboard to (read clip_out.png as «class PNGf») PREV=$(clipboard info | md5)
STEP 06

알림 표시

압축률과 전후 용량을 macOS 알림으로 표시합니다. 썸네일로 압축된 이미지를 미리 확인할 수 있습니다.

terminal-notifier \ -title "ClipComp" \ -message "84% 완료" \ -contentImage \ clip_out.png
1초 후 STEP 01로 반복
임시 파일은 매 루프마다 삭제됩니다. 스크립트 종료 시 trap EXIT으로 최종 정리됩니다.
